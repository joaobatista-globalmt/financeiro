### 4. Fix do dropdown que não abria no click (commit `691178f`)

**Sintoma reportado:** o botão `[ + Novo ▾ ]` aparecia mas o menu não abria
ao clicar.

**Diagnóstico:**
1. CSS do dropdown só funcionava via `:hover` (não dispara em mobile/tablet,
   e em alguns trackpads é instável)
2. O primeiro upload do CSS no commit `e3993ed` **não tinha sido aplicado
   completamente** — race condition no write stream do SFTP (handler `close`
   dispara antes do flush completo no disco)
3. JS não tinha cache buster e CSS estava em `?v=2` antigo

**Solução:**
- `financeiro.js`: novo handler de `click` que adiciona classe `.open` ao
  `.dropdown` (fecha outros abertos, fecha em click fora, fecha com ESC)
- `financeiro.css`: regra `.dropdown:hover .dropdown-menu` ampliada para
  `.dropdown:hover .dropdown-menu, .dropdown.open .dropdown-menu`
- `layout/header.php`: cache buster `?v=2` → `?v=3`
- `layout/footer.php`: cache buster no JS `?v=2`

**Arquivos:**
- `public/assets/financeiro.css`
- `public/assets/financeiro.js`
- `src/views/layout/header.php`
- `src/views/layout/footer.php`

**Validação:** curl retorna HTML com `?v=3` no CSS e `?v=2` no JS. CSS tem
3 ocorrências de `page-action` e regra `.dropdown.open .dropdown-menu` ativa.
